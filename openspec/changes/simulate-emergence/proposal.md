# 提案：模拟涌现 (Simulate Emergence)

## 背景
用户需要一个能支撑 10w+ 粒子的颜色粒子涌现效果模拟器。
这不仅是一个视觉 demo，更是一个高性能计算挑战。
仅仅能跑是不够的，必须跑得快，跑得优雅。

## 目标
在 `emergence-sim` 文件夹中创建一个全新的 Web 应用。
实现基于网格（Grid-based）的粒子交互系统。
支撑 10w+ 粒子流畅运行 (60fps)。

## 核心变更
1. **新建项目**：初始化 `emergence-sim` (Vite + Vanilla TypeScript)。
2. **核心引擎**：实现基于 `Float32Array` (SoA) 的粒子系统。
3. **空间索引**：实现基于网格的高效邻域搜索算法。
4. **渲染管线**：使用 WebGL 进行高性能渲染（拒绝 10w 个 DOM 节点）。

## 理由
- **性能决定一切**：10w 粒子意味着任何对象分配都会导致 GC 抖动。必须使用 Data-Oriented Design。
- **网格优于树**：在粒子均匀分布的涌现模拟中，不仅实现简单，而且缓存友好。
