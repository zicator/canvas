# 提案：AI 图像生成画布 (AI Canvas)

## 背景
用户计划基于开源项目 `tldraw` 构建一个 AI 图像生成画布应用。
目标是允许用户直接在无限画布上调用 AI 生成图片。
这是一个长期迭代的需求，目前首先需要提供一个模拟接口（Mock），将预设的占位图返回到画布上，以便优先验证交互流程。

## 问题陈述
- 我们需要一个可视化的 AI 图像生成界面。
- `tldraw` 提供了强大的无限画布功能，但需要与 AI 工具进行深度集成。
- 真实的 AI API（如 SD/Midjourney）调用成本高且响应慢，我们需要一个"模拟模式"来快速迭代前端 UI/UX。

## 解决方案
基于 `tldraw` SDK 创建一个新的 Web 应用（或模块）。
引入自定义的 "AI 生成" 工具/形状。
- **阶段 1 (本次范围)**:
    - 搭建 `tldraw` 开发环境。
    - 创建自定义交互形状 "AI Image Frame"。
    - 实现 "Mock Provider" 模拟网络延迟并返回占位图。
    - 定义请求/响应接口，为未来接入真实 API 做好标准化设计。

## 范围 (Scope)
- **包含**:
    - 项目初始化 (React + tldraw)。
    - 自定义 Shape 定义 (UI + 逻辑)。
    - Mock Service 实现。
    - 画布 UI 集成 (工具栏/上下文菜单)。
- **不包含**:
    - 真实的 Stable Diffusion/Midjourney 集成 (规划在阶段 2)。
    - 后端服务 (目前仅做客户端 Mock)。
    - 用户认证/管理系统。

## 风险
- `tldraw` 自定义 API 的复杂度（文档可能滞后）。
- Mock 接口设计是否能完美覆盖未来真实 API 的特性（如流式传输、进度更新）。
