# 设计：自动选中新生成的画板

## 架构
无架构变更。这是 `MainLayout` 组件内的逻辑增强。

## 实现细节

### `MainLayout.tsx`

在 `handleGenerate` 函数中：

1.  **当前流程**:
    - 计算位置。
    - 创建画板形状 (`editor.createShape`)。
    - 创建占位符。
    - 视口逻辑。
    - 调用服务。

2.  **新流程**:
    - 计算位置。
    - 创建画板形状。
    - **动作**: `editor.select(boardId)`
    - 创建占位符。
    - 视口逻辑。
    - 调用服务。

## 边界情况
- **多次生成**: 如果用户再次生成，*新*的画板应该被选中（自动取消选中旧的）。`editor.select` 默认会替换当前选择，因此可以自动处理。
- **视口动画**: 选中操作应在视口动画之前或期间发生。这不会造成冲突。
